# VISUAL SYSTEM

## Philosophy

This portfolio should feel less like a designer portfolio and more like a carefully curated digital exhibition.

The experience should communicate:

- Precision
- Restraint
- Confidence
- Clarity
- Craft

The visual language sits between:

- Swiss Design
- Editorial Design
- Architecture
- Contemporary Art Direction

References include:

- Studio Now
- Olaf
- Jonas (portfolio reference)
- Apple Product Marketing
- Dieter Rams
- Jony Ive
- Acne Studios
- COS
- Frama
- Aesop

The work should never feel trendy.

Avoid:
- Startup aesthetics
- SaaS aesthetics
- Dribbble aesthetics
- Bright gradients
- Glassmorphism
- Neumorphism
- Excessive animations
- Generic portfolio patterns

The design should feel timeless.

---

# Color System

Primary Background

#F1F1F1

Primary Text

#070707

No pure black.
No pure white.

The site should rely on contrast through typography, scale, spacing and hierarchy rather than color.

Accent colors should only exist if a project requires them.

Outside project pages:

Use only:

#F1F1F1
#070707

---

# Typography

Primary Typeface:

PP Mori

Typography should carry most of the visual identity.

The interface should feel typographic first.

Large typography is encouraged.

Headlines should feel bold and architectural.

Avoid decorative typography.

Avoid mixing multiple font families.

Use:

PP Mori Regular
PP Mori Medium

Only.

Avoid:
- Thin
- Extra Light
- Black

---

# Layout Principles

The layout should feel designed by a senior designer.

Every section should have intention.

Use:

- Large margins
- Strong whitespace
- Generous breathing room

Content should never feel compressed.

Every block should feel curated.

When in doubt:

Remove.

Do not add.

---

# Grid System

Use a strict grid.

Desktop:

12 columns

Tablet:

8 columns

Mobile:

4 columns

Alignment should feel mathematical.

Avoid random positioning.

Avoid floating decorative elements.

---

# Imagery

Images should feel editorial.

Large imagery is preferred.

Allow imagery to dominate sections.

Images should breathe.

Avoid:
- Tiny image cards
- Excessive borders
- Drop shadows
- Decorative frames

Images should sit naturally inside the layout.

---

# Motion

Motion should feel invisible.

Animations exist only to support understanding.

References:

Apple
Linear
Notion

Use:

- Fade
- Scale
- Subtle translate

Avoid:

- Bounce
- Elastic effects
- Spin
- Flashy reveals

Motion duration:

200ms–500ms

Maximum.

---

# Components

Components should disappear.

The user should notice the content.

Not the UI.

Avoid:

- Fancy cards
- Decorative containers
- Unnecessary backgrounds

Every component should justify its existence.
